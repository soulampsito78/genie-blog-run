# today_genie TPO proof artifacts (as implemented)

This file describes **only** what `scripts/run_tpo_proof_once.py` does in this repository. It is not a product roadmap.

## Entrypoint

- **Command:** from repository root, `python3 scripts/run_tpo_proof_once.py` (or `python` if available).
- **Feeds:** reads JSON from `ops/feeds/` per `_FEED_FILES` in that script; exits non‑zero if any required file is missing or empty.
- **Pipeline:** `build_runtime_input("today_genie")` → `build_full_prompt` → `call_gemini` → `parse_model_json` → `finalize_today_genie_hashtag_list` → `validate_today_genie` → (if not blocked) image generation → `_proof_html_body` → `_write_proof_bundle`.

## Artifact set (per run)

All under `output/` unless otherwise noted. **`stamp`** = KST local `MMDD_HHMM` from `_today_genie_local_stamp`.

| Artifact | Pattern |
|----------|---------|
| Preview JSON | `today_genie_preview_{stamp}.json` |
| Proof HTML | `today_genie_tpo_proof_{stamp}.html` |
| Operator log | `today_genie_tpo_proof_{stamp}.operator.txt` |
| Top image | `GENIE_EMAIL_today_genie_top_tpo_run_{stamp}.jpg` |
| Bottom image | `GENIE_EMAIL_today_genie_bottom_tpo_run_{stamp}.jpg` |

Reference image (input, not output): `static/email/GENIE_REF_today_genie_master_v1.jpg`.

## Success-mode proof HTML (`failure_mode=False`)

Reader-facing sections, in order:

1. `1. 브리핑 제목` — title box  
2. Top hero image — **only** if `_proof_top_hero_html` returns non-empty (`image_run.top.generated_this_run == True`); reader HTML uses no status caption unless debug path  
3. `2. 핵심 요약`  
4. `3. TOP 3 뉴스 브리핑`  
5. `4. 리스크 요약`  
6. `5. 해시태그` — from `_proof_hashtag_html(..., omit_muted_label=True)` or muted “해시태그 없음”  
7. `6. 최종 판정` — includes TPO+룰렛+이미지 게이트를 요약한 `final_pass` 결과  

Optional trailing block: **`TODAY_GENIE_PROOF_OPERATOR_DEBUG`** enabled → appended `operator_debug_html` (운영·디버그 sections including image direction, bottom slot HTML, TPO/roulette dumps, truncated `runtime_input` JSON).

## Failure-mode proof HTML (`failure_mode=True`)

Used for all `_write_fail_proof` paths (feeds incomplete, validation block, HTTP/content exceptions, missing image prompts, etc.). Reader-facing sections, in order:

1. Draft notice box: 게시 확정 실패 · 참고용 초안  
2. `1. 브리핑 제목(참고)`  
3. `2. 핵심 요약(참고)`  
4. `3. TOP 3 뉴스 브리핑(참고)`  
5. `4. 리스크 요약(참고)`  
6. `5. 핵심 사유 요약` — sanitized reasons  
7. `6. 최종 판정`  
8. One line: `미리보기 JSON: <path>`  

No top hero in failure reader HTML. Same optional `operator_debug_html` when **`TODAY_GENIE_PROOF_OPERATOR_DEBUG`** is set.

## Operator vs reader separation

- **Default:** Raw `runtime_input` JSON is **not** embedded in reader-facing proof HTML.
- **`today_genie_tpo_proof_{stamp}.operator.txt`:** written by `_write_operator_diagnostic_log` with errors, raw reasons, full `runtime_input` JSON dump.
- **Reader HTML:** debug dump only when **`TODAY_GENIE_PROOF_OPERATOR_DEBUG`** is truthy (`1` / `true` / `yes`).

## Environment variables (relevant)

| Variable | Role |
|----------|------|
| `PROJECT_ID` | Vertex project (required for text/images via `main`) |
| `VERTEX_LOCATION` | Vertex region |
| `VERTEX_MODEL` | Text model (from `main`) |
| `VERTEX_IMAGE_MODEL` | Image model default `gemini-2.5-flash-image` if unset |
| `TODAY_GENIE_PROOF_OPERATOR_DEBUG` | Append operator/debug sections to proof HTML |
| Feed envs | Script sets `TODAY_GENIE_*_JSON` from `ops/feeds/` files |

## Legacy artifacts

Historical proof/preview files from before single-source alignment may live under `output/archive/legacy_tpo_pre_alignment/`. They are **not** regenerated by current code paths and must not be used as the definition of current layout.

## Validation gates (today_genie)

- `validate_today_genie` **`pass`**: 풀 파이프라인(이미지 포함)으로 진행 가능.
- **`block`**: 이미지 미생성, `_write_fail_proof` 로 실패 증빙 HTML.
- **`draft_only`** (경고만): 로컬 TPO 스크립트는 게시 확정 불가로 처리 — 이미지 미생성, 동일 실패 증빙 경로, 프로세스 종료 코드 **7**.

실패 증빙 HTML 하단에는 운영자 편의로 **미리보기 JSON 상대경로 한 줄**만 두며, 원시 `runtime_input` 전체는 `.operator.txt` 에만 있다.
